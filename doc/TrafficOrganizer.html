class TrafficOrganizer


The TrafficOrganizer class is intended to expand the functionality of Omniture's limited traffic source reporting capabilities. The class has been designed in mind for use with Omniture SiteCatalyst, but could theoretically be used for other web analytics tools. At it's core, the TrafficOrganizer borrows heavily from Google Analytics traffic source reporting functionality but also adds a number of enhancements (e.g. referrer groups, keyword groups). At it's core, the traffic source manager looks at the following dimensions to classify all incomming traffic to a web site:

  • medium/channel
  • source
  • keyword
  • keyword group
  • content
  • campaign name
  • referring domain
  • referring path
For ease of use, traffic source manager also makes the full referrer and referring domain available.
Version: 1.0

Author: Mike Sukmanowsky

track

void track()
    The core function of the traffic source manager class intended to be used after an instance is initialized and configured. The logic of this function is somewhat complex:
    1. If no traffic source cookie is found, the traffic source will be classified.
    2. If a traffic source cookie is found, it may be overwritten if and only if:
      1. The referrer of the current page does not match the list of ignored referrers OR
      2. The medium and source query string parameters are present and have values set
